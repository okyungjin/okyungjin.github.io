# dvlog 글쓰기·리뷰 가이드

dvlog 블로그 글을 작성하거나 리뷰할 때 반드시 지켜야 하는 규칙을 모은 문서입니다.
(한글, 규칙은 계속 추가됩니다.)

## 표기 규칙

### 1. 특수문자 대시 사용 금지

문장 부호로 em dash(`—`)나 en dash(`–`)를 쓰지 않습니다. 대신 **쉼표, 괄호, 콜론(`:`), 또는 줄바꿈**으로 대체합니다.

- 나쁜 예: `gh-ost — GitHub이 만든 도구`
- 좋은 예: `gh-ost, GitHub이 만든 도구`
- 좋은 예: `gh-ost(GitHub이 만든 도구)`
- 좋은 예: `gh-ost: GitHub이 만든 도구`

목록 항목에서 `용어 — 설명` 형태도 `용어: 설명`으로 적습니다.

> 참고: `INT → BIGINT`처럼 의미가 분명한 기술적 화살표 표기는 허용합니다. 금지 대상은 문장 부호로 쓰이는 대시(`—`, `–`)와 장식용 특수문자입니다.

## 작성·리뷰 체크리스트

리뷰 시 아래 항목을 확인합니다.

- [ ] em dash(`—`)·en dash(`–`) 등 금지 특수문자가 없는가 (규칙 1)
- [ ] frontmatter에 `title`, `description`, `pubDate`, `category`, `tags`가 모두 채워졌는가
- [ ] `category`가 사이드바 카테고리 체계와 맞는가 (예: `MySQL`, `PostgreSQL`)
- [ ] 코드블록에 언어가 지정되어 있는가 (` ```sql `, ` ```bash ` 등)
- [ ] 제목 위계(h2/h3)가 자연스럽고 TOC가 보기 좋게 잡히는가
- [ ] 외부 자료를 인용했다면 출처 링크를 남겼는가
